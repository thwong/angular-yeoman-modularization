Angular-yeoman-modularization

angular-yeoman generator based template with modularization

View on GitHub Download .zip Download .tar.gz

Why a new template?

While there are many angular based templates out there, not many of them are optimized for building large scale web applications that contain and share a large set of components. The angular generator created and maintained by the yeoman team is one of the best templates out there; however, even so, it does not contain built-in supports for capabilities such as lazy-loading, i18n supports, code coverage integrations, etc. which are keys for building and maintain large scale applications. This is the reason why I introudce this new template that contains all the building blocks weird in and ready for use.

Using yeoman generator as base.

The basic structure of this template is generated using the Yeoman Angular Generator. Therefore, you can utilize the nice generator functions to creating new controllers, services, directives, etc. Here are some examples:

Here are some examples: (You can view the complete list on the Yeoman website)


> yo angular:controller Main           // Create a new controller
> yo angular:directive FileUpload      // Create a new directive
> yo angular:factory User              // Create a new factory

Using Coffeescript as main programming language

This template uses Coffeescipt as the main programming language which will be compiled into Javascript by the grunt watcher. There are multiple main benefits of using coffeescript:

  • The grunt coffeescript watcher compiles the code on demand, which helps developers to catch code errors early
  • The coffeescript syntax is concise and less-verbose, which helps developers to focus on the core business logic
  • The complied Javascript codes are robust and following many Javascript best practices

Using LESS as the main styling language

This template uses LESS as the main styling language which will be complited into CSS by the grunt watcher, and the organization of the LESS files follow the SMACSS pattern. There are multiple main benefits of using LESS:

  • The grunt less watcher compiles the LESS code on demand, which helps developers to view the style change instantly
  • The SMACSS guide file structure helps developers to maintain CSS class dependency and modularization

Using grunt and bower for build

This template uses Grunt as the build system that handles the compilation of files, packaging of contents, CSS and image optimization, test runners, etc. The dependency of external plugins are done using bower.

Modulizing codes wiht UI router and ocLazyLoader

To maintain efficent loading time of all pages, this template uses the ocLazyLoader with the Angular UI-Router to load in only files that are required for any particular route. This solves one of the main problems that one faces when using the original yeoman generator - that is all the files and dependencies are loaded up front. When the application grows, the amount of code loaded in up front is extremely large.

Providing i18n capabilities via getText

To make i18n as painless and as easy as possible, this template uses the getText component. There are couple main benefits of using getText:

  • Easy to get started. Developer can start with the English version of the application without worrying about any translation at all.
  • With the translate directive, it is very easy for developer to annoate all strings that need to be translated.
  • Translate of new or exisitng strings made extremely simple with the Poedit editor
